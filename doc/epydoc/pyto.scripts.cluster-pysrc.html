   1  #!/usr/bin/env python 
   2  """ 
   3  Makes and analyzes clusters of (already existing) boundaries and segments. 
   4   
   5  For example, in case of presynaptic terminal, vesicles are boundaries and 
   6  connectors are segments. 
   7   
   8  Clustering can be done based on connectivity (see "Clustering by connectivity" 
   9  parameter section) or on distance (see "Hierarchical clustering of connections" 
  10  and "Hierarchical clustering of boundaries" parameter sections). 
  11   
  12  In short, clustering based on connectivity makes: 
  13    - clusters of boundaries that are connected (linked) by segments  
  14    - clusters of segments connected by boundaries.  
  15  Clustering based on distance (hierarchical clustering) makes: 
  16    - clusters of boundaries 
  17    - clusters of segments 
  18  In addition, clusters dual to distance based clusters are also made  
  19  (experimental feature): 
  20    - clusters of segments that contact distance boundary clusters (dual to 
  21    distance boundary clusters) 
  22    - clusters of boundariess that contact distance segment clusters (dual to 
  23    distance segment clusters) 
  24   
  25  For more info how these clusters are made see class pyto.scene.MultiCluster. 
  26   
  27  Important notes: 
  28   
  29    - In order to make setting up multiple scripts easier, parameters common  
  30  to these scripts are expected to be read from tomo_info.py file. The location  
  31  of this file is given as argument path of common.__import__(). These parameters 
  32  are set in this script in lines having the following form: 
  33   
  34    if tomo_info is not None: labels_file_name = tomo_info.labels_file_name 
  35   
  36  Specifiying another value for the same parameter (here labels_file_name)  
  37  overrides the value from tomo_info.
