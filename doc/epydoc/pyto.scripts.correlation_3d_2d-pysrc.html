  2  """ 
  3  Establishes a correlation between 3D and 2D coordinate systems (e.g. images)  
  4  and correlate the position of objects of interest from the 3D to the 2D system.  
  5   
  6  Typically the initial (3D) system is a light micrroscopy (confocal) image 
  7  and the final (2D) is a ion beam image. The correlation procedure used in this  
  8  script is direct, meaning there are no intermediary systems.  
  9   
 10  Conceptually, the correlation procedure consists of two steps: 
 11    1) Establish a correlation between the two systems (initial and final),  
 12  that is find a coordinate transformation between the two systems. 
 13    2) Correlate positions of obects of interest from one system to the other. 
 14   
 15  Marker points are needed to determine the transformation  (step 1). That is,  
 16  one needs to identify one set of features that are visible in both systems. 
 17  The coordinate transformation between the two systems is based on the full 3D  
 18  rigid body transformation (rotation, scale and translation). Marker positions  
 19  are specified as 3D coordinates in the initial system and 2D coordinates  
 20  (z component is missing) in the final system. 
 21   
 22  Requires the following coordinates: 
 23    - marker coordinates in the initial (3D) system 
 24    - corresponding marker coordinates in the final (2D) system 
 25    - coordinates of objects of interest in the initial system 
 26   
 27  These coordinates have to be saved in a file, in a table where rows represent 
 28  individual points (markers and objects of interest) and (some of) columns 
 29  contain x, y and z coordinates. 
 30   
 31  It is not necessary, but it is recommended to use ImageJ / Fuji to generate 
 32  these coordinates. What follows is the procedure which is compatible with 
 33  the default values in the File format sectin of the Parameters (below): 
 34   
 35    1) (optional) Set to pixels: ImageJ / Image / Properties: Units = pix;  
 36       x width = y width = 1 
 37   
 38    2) Open measurement options by Analyze / Set measurements 
 39   
 40    3) Set measurements: mean grey and stack position are sufficient;  
 41       display label, add to overlay, decimal places=1 are useful 
 42   
 43    4) To store selected points to ROI (useful): Edit / Options / Point tool  
 44       (or double click on point tool): Auto measure, add to ROI, label 
 45   
 46    5) Activate point tool 
 47   
 48    6) For each point (markers in both systems and objects of interest): Click  
 49       on a point (if auto measure was not set need Ctrl-M to put it in  
 50       the results; shift-click might also work) 
 51   
 52    7) When ROI manager opens or Analyze / Tools / ROI manager check  
 53       ROI manager / More / Options / Associate ... with slices in order that  
 54       points are shown only on the corresponding slices  
 55   
 56    8) Save results: Results / File / Save as. The name of this file 
 57   
 58    9) Save ROIs (in zip format) : ROI manager / More / Save 
 59   
 60  Steps 4, 7 and 9 are useful because they allow saving ROIs and retrieving  
 61  them at alater point, but are not strictly necessary. Picks saved at ROIs  
 62  can be displayed on the image at a later point but it's hard to read the  
 63  pick coordinates. On the contrary, the coordinates are easily accessible  
 64  in the results file, but it is difficult to display the picks on 
 65  the same or on another image. 
 66    
 67  Standard usage: 
 68   
 69  1) Edit this file to enter the desired parameters (coordinates of markers  
 70  and objects of interest, and the transformation type) 
 71  2) Import and run this file (in IPython or another python shell): 
 72    >>> import correlation_3d_2d 
 73    >>> correlation_3d_2d.main() 
 74   
 75  Advanced usage: 
 76   
 77  1) As in the standard usage. 
 78  2) Import this file and execute commands from main() one by one. Check values  
 79  of variables as needed: 
 80    >>> import correlation_3d_2d 
 81    >>> from correlation_3d_2d import *
