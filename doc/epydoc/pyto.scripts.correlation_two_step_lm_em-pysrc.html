  1  #!/usr/bin/env python 
  2  """ 
  3   
  4  Establishes a correlation between a light microscopy (LM) image and EM stage  
  5  coordinates and correlates positions of objects of interest (targets) between  
  6  EM and LM.  
  7   
  8  In order to facilitate establishing a correlation between LM and EM stage, 
  9  EM overview is added as an intermediate system. Consequently, this procedure  
 10  involves three coordinate system: 
 11    - LM: LM image 
 12    - EM overview: (relativly) low magnification EM image   
 13    - EM stage (also called search): EM stage coordinate system 
 14   
 15  Conceptually, the correlation procedure consists of the following steps: 
 16    1) Determine transformation between LM and EM overview systems. 
 17    2) Determine transformation between EM overview and EM search systems. 
 18    3) Calculate transformation between LM and search systems. 
 19    4) Correlate spots (of interest) specified in one system to other systems. 
 20   
 21  To establish a correlation between two systems means to find a coordinate  
 22  transformation between the two systems. 
 23   
 24  Marker points are needed for both transformation determinations  
 25  (steps 1 and 2). That is, one needs to identify one set of features that are  
 26  visible in both LM and EM overview images and another, and another set of  
 27  EM overview features that are generally smaller and closer to spots  
 28  of interest. 
 29   
 30  Requires the following coordinates: 
 31    - marker coordinates in the LM system 
 32    - marker coordinates in the EM stage system 
 33    - marker coordinates in the EM overview system that orrespond to the  
 34      LM markers 
 35    - marker coordinates in the EM overview system that orrespond to the  
 36      EM stage markers 
 37    - (optional) target coordinates the LM or EM stage systems 
 38   
 39  In this script, coordinates of markers and targets in LM and EM overview  
 40  systems are specified by a file, while EM stage coordinate are entered  
 41  directly (because they are usually read from the microscope). A coordinate 
 42  file can be generated by clicking on targets in ImageJ. Other formats 
 43  can be used as long as the data is in a table format where rows correspond to  
 44  different markers and targets and the x, y, ... coordinates are specified by  
 45  columns. The calculated transformation parameters and the coordinates of the  
 46  correlated target points are written in a results file.  
 47   
 48  Standard usage: 
 49   
 50  1) Edit this file to enter the desired parameters (coordinates of markers  
 51  and objects of interest, and the transformation type) 
 52  2) Import and run this file (in IPython or another python shell): 
 53    >>> import correlation_two_step_lm_em 
 54    >>> correlation_two_step_em_lm.main() 
 55   
 56  Advanced usage: 
 57   
 58  1) Edit this file to enter the desired parameters (coordinates of markers  
 59  and objects of interest, and the transformation type) 
 60  2) Import this file and execute commands from main() one by one. Check values  
 61  of variables as needed: 
 62    >>> import correlation_two_step_lm_em 
 63    >>> from correlation_two_step_lm_em import * 
 64        ... 
 65  The main object (corr) is an instance of pyto.scene.EmLmCorrelation. It has   
 66  attributes for each individual transformation (corr.lm2overview,  
 67  corr.overview2search and corr.lm2search) that are instances of  
 68  pyto.geometry.Affine2D. Please check docs for these classes for attributes 
 69  and methods.  
 70   
 71  The details of the complete procedure (summarized above) are as follows: 
 72   
 73    1) Find transformation between LM and EM overview systems using specified 
 74    markers. LM markers are typically specified as (coordinates of) features on a  
 75    LM image, and overview markers (as coordinates) of the same features on a  
 76    EM overview image. 
 77   
 78    The EM overview image can be a low mag (such as 220x) EM image, so that  
 79    the whole grid square fits on this image. However, a large image shift  
 80    between this low mag and the high mag used for the stage system can 
 81    be a problem. If this is the case, the overvew image can be a collage  
 82    (mosaic) of medium mag images (1-5 kx, for example), such as that obtained  
 83    by SerialEM.  
 84   
 85    This transformation is an affine transformation in 2D, that is it is composed 
 86    of a Gl (general linear) transformation and a translation. The Gl  
 87    transformation can be decomposed into rotation, scaling along two principal  
 88    axes, parity (flipping one axis) and shear. The LM - overview transformation 
 89    can be calculated in two ways: 
 90   
 91      (a) Direct with single overview image: Markers lm_markers and  
 92      overview_markers need to correspond to each other, the transformation  
 93      is calculated directly. The overview image can be a single or a mosiac 
 94      image. 
 95   
 96      (b) Separate gl and translation:  Markers lm_markers_gl and  
 97      overview_markers_gl have to outline the same shape in the same orientation 
 98      but they need not be the same spots, that is they can have a  
 99      fixed displacement. For example, holes on a quantifoil can be used for this  
100      purpose. These parameters are used to find the Gl transformation. In the  
101      next step, parameters lm_markers_d and overview_markers_d are used to  
102      calculate only the translation.  
103   
104    2)  Find transformation between EM overview and EM search systems using  
105    (overview and search) markers (here called details). The transformation is 
106    also affine, but it can be restricted so that instead of the full Gl  
107    transformation only orthogonal transformation is used (rotation, one scaling  
108    and parity). EM overview images have to have the same mag as the one used  
109    for the LM - overview transformation, while the search system can be chosen  
110    in a different way: 
111   
112      (a) Move search: The same overview image (single or mosaic image)  
113      as the one used for the LM - overview transformation is used for the  
114      overview system, but the stage movement system is used for the search  
115      system. Specifically, for each detail (markers for this transformation)  
116      found on the overview image, the EM stage needs to be moved so that the  
117      same feature is seen in the center of the EM image made at a medium mag  
118      (image size typically up to 10 um). The stage coordinates are used as  
119      search details. Parameter overview2search_mode has to be set to  
120      'move search'. The difficulty here is to find enough features that are  
121      seen in the overview image but can be easily navigated to in search  
122      (cracks in ice are often used). 
123   
124      (b1) Move overview, single overview image: First one feature  
125      needs to be identified on the overview image used for the LM - overview  
126      transformation (main overview image). The coordinates of that feature  
127      are used as one marker (search_detail) and the EM stage cooordinates  
128      for that image is the corresponding search marker (search_detail).  
129      This particular stage position has also to be specified as search_main  
130      parameter. Then, additional overvew images have to be obtained under  
131      the same imaging conditions (magnification, size, binning) as was the  
132      case for the main overview image, but at different stage positions.  
133      These state positions have to be such so that the feature detected on  
134      the main overview image is present on the overview images (typical  
135      movements are 10 - 20 um). The other overview markers (details) are  
136      coordinates of the same feature on the additional overview images,  
137      while the corresponding stage coordinates are used as overview and  
138      stage markers (details). Parameter overview2search_mode has to be set  
139      to 'move overview'. This is perhaps an easier method to use than the 
140      previous one, but it is conceptually moredifficult. 
141   
142      (b2) Move overview, mosaic overview image. This is a variant of the  
143      previous method (b1) and everything already stated for b1 applies 
144      here also. In this case, parameter mosaic has to be True. Also, in  
145      addition to the mosaic image, a single image (overview EM image) of  
146      the same magnification (and binning, if used for mosaic) has to be  
147      specified. A single feature has to be marked on the mosaic and the  
148      EM overview images in order to provide correlation between these two s 
149      ystems (parameters overview_mosaic_main_row and overview_main_row).  
150   
151    3) Calculate transformation between LM and search systems as a composition of 
152    the LM - overview and overview - search transforms 
153   
154    4) Correlate spots specified in one system to the other systems. Coordinates  
155    of spots correlated to search system are interpreted according to the mathod 
156    used to establish the overview - search transformation (see point 2) 
157   
158      (a) Collage: Spots in search system are simply the coordinates in the  
159      collage used for the overview - search transformation. 
160   
161      (b) Stage, move search: Correlated spots are stage coordinates where 
162      spots are located in the center of search images (medium mag). 
163   
164      (c) Stage, move overview: Correlated spots are stage coordinates. An search 
165      image made at this stage position (low mag) contains the spot at the 
166      coordinate specified by parameter overview_center. 
167   
168  Note: For reading rows from a file (such as lm_markers_file) top rows 
169  containing comments are ignored, and the data rows are numbered from 0 up.  
170  That is, the first data row is specified by row 0 in parameters such as  
171  lm_markers_rows.

581              specs=mosaic_read, format=positions_file_type, xyColumns=xy_columns) 
582          corr.overviewMarkers = ( 
583              corr.overviewMarkers - corr.mosaic_markers[0]  
584              + corr.mosaic_markers[1]) 
585   
586      ##################################################################### 
587      # 
588      # Find transformation between EM overview and EM stage systems 
589      # 
590     
591      # read overview detail positions 
592      corr.mode = overview2search_mode 
593      pos_read = {} 
594      pos_read['overviewDetail'] = ( 
595          overview_detail_file, overview_detail_rows) 
596      corr.readPositions( 
597          specs=pos_read, format=positions_file_type,  
598          xyColumns=overview_detail_xy_columns) 
599   
600      # read search detail positions 
601      pos_read = {} 
602      try: 
603          corr.searchDetail = numpy.asarray(search_detail) 
604      except NameError: 
605          pos_read['searchDetail'] = (search_detail_file, search_detail_rows) 
606          corr.readPositions( 
607              specs=pos_read, format=positions_file_type,  
608              xyColumns=search_detail_xy_columns) 
609   
610      # setup move overview mode for overview2search 
611      if corr.mode == 'move overview': 
612          corr.searchMain = search_main 
613          corr.overviewCenter = overview_center 
614   
615      # setup spots 
616      spot_pos_read = {} 
617      try: 
618          spot_pos_read['lmSpots'] = (lm_spots_file, lm_spots_rows) 
619      except NameError:  
620          corr.lmSpots = None 
621      corr.readPositions(specs=spot_pos_read, format=positions_file_type,  
622                         xyColumns=lm_spots_xy_columns) 
623      try: 
624          corr.overviewSpots = numpy.asarray(overview_spots) 
625      except NameError:  
626          corr.overviewSpots = None 
627      try: 
628          corr.overviewSpotLabels = overview_spot_labels 
629      except NameError:  
630          corr.overviewSpotLabels = None 
631      try: 
632          corr.searchSpots = numpy.asarray(search_spots) 
633      except NameError:  
634          corr.searchSpots = None 
635          try: 
636              spot_pos_read['searchSpots'] = (search_spots_file,  
637                                              search_spots_rows) 
638          except NameError: pass 
639      try: 
640          corr.searchSpotLabels = search_spot_labels 
641      except NameError:  
642          corr.searchSpotLabels = None 
643      corr.readPositions(specs=spot_pos_read, format=positions_file_type,  
644                         xyColumns=search_spots_xy_columns) 
645   
646   
647      ##################################################################### 
648      # 
649      # Calculate transformation between LM and search systems 
650      # 
651               
652      # establish correlation 
653      corr.establish(lm2overviewType=lm2overview_type,  
654                     overview2searchType=overview2search_type) 
655   
656      # inverse correlations 
657      corr.search2lm = corr.lm2search.inverse() 
658      corr.overview2lm = corr.lm2overview.inverse()  
659      if corr.overview2lm.rmsError is None: 
660          scale = numpy.sqrt(numpy.multiply.reduce(corr.overview2lm.scale)) 
661          if corr.lm2overview.rmsError is not None: 
662              corr.overview2lm.rmsErrorEst = corr.lm2overview.rmsError * scale 
663          elif corr.lm2overview.rmsErrorEst is not None: 
664              corr.overview2lm.rmsErrorEst = corr.lm2overview.rmsErrorEst * scale 
665          else: 
666              corr.overview2lm.rmsError = None 
667      corr.search2overview = corr.overview2search.inverse()  
668   
669      ##################################################################### 
670      # 
671      # Correlate spots (of interest) specified in one system to other systems 
672      # 
673               
674      # correlate LM spots 
675      if corr.lmSpots is not None: 
676          corr.searchFromLmSpots = corr.lm2search.transform(corr.lmSpots) 
677          corr.overviewFromLmSpots = corr.lm2overview.transform(corr.lmSpots) 
678   
679      # correlate overview spots 
680      if corr.overviewSpots is not None: 
681          corr.lmFromOverviewSpots = \ 
682              corr.overview2lm.transform(corr.overviewSpots) 
683          corr.searchFromOverviewSpots = \ 
684              corr.overview2search.transform(corr.overviewSpots) 
685   
686      # correlate EM search spots 
687      if corr.searchSpots is not None: 
688          corr.lmFromSearchSpots = corr.search2lm.transform(corr.searchSpots) 
689          corr.overviewFromSearchSpots = \ 
690              corr.search2overview.transform(corr.searchSpots)

692      #####################################################################

694      # Output 
695      #
