Sets attributes from arguments and reads serieal em (mdoc) file.

If arg aligned prefix is None, it is set to the default values for dm ('') and dimi alignments ('sum.'). If ucsf alignment is specified (arg alignment) aligned prefix has to be given.

Arguments:

  • serialem_dir: directory where serial em mdoc and stacks are
  • orig_frames_dir: directory where original (unaligned) frames are
  • aligned_frames_dir: directory where aligned frames are located
  • align: alignment program used ('dm', 'dimi' or 'ucsf')
  • aligned_prefix: prefix added to (unaligned) frames to make aligned frames file names
  • ucsf_translations_path: file path for the ucsf alignment log file (sent to stdout by ucsf alignment program)
  • apixel: pixel size in A
  • counte: number of counts per electron (can be found in the header of original frames)

Attributes set:

  • all arguments are saved as attributes with same names
  • self.serialem: SerialEM object
  • self.dm_aligned_prefix = "sum."
  • self.dm_translation_prefix = "Measured drift for "
  • self.aligned_path_prefix: self.aligned_frames_dir / self.aligned_prefix
  • self.seriealem: (SerialEM) SerialEm object
  • self.apixel: pixel size in A
  • self.counte: counts per electrons

projections(self, projections=None, serialem_stack=False, stats=True, translations=False, names_only=False)

source code 

Generator that yeilds projection images. At each iteratition one or more images corresponding to the current tilt angels are yielded.

The following file / directory name conventions have to be followed:

  • In Dimi alignment the frame stacks and the aligned projection images have the same file names (but they reside in different directories)
  • In DM alignment the aligned projection file names are obtained by adding prefix 'sum' (more precisely self.dm_aligned_prefix) to the corresponding frame stacks
  • In DM alignment the thanslation file names are obtained by adding prefix 'Measured drift for ' (more precisely self.dm_translation_prefix) to the corresponding frame stacks
  • In UCSF alignment the aligned projection file names have a user specified prefix

The yielded images are selected in the following way:

  • All serialEM log (mdoc) files residing in self.serialem directory are combined (they should all correspond to the same tilt series)
  • Names (without directory path) of the projection frame stacks (each image is a stack of frames) and tilt angles are read from serialEM mdoc file(s) and sorted by tilt angle.
  • SerialEM projections are read from seriealEM projection stacks that are located in self.serialem directory and correspond to the mdoc files
  • Complete paths to the projection frame stacks are obtained from self.orig_frames_dir and their names
  • Projection frame stacks are flattened (along z-axis) to non-frame-aligned projection images
  • Complete paths to frame aligned projection images are obtained from self.aligned_frames_dir and the projection frame stack names
  • Paths to files containing frame alignment translations are determined from frame aligned projection file names (for Dimitri and DM alignment implemetnations)

In case has not been set already (self.apixel is None), it is set to the value reported in mdoc file.

If self.counte is None and frames stacks are in mrc format, self.count set to the value given in the frames stacks header.

Arguments:

  • serialem_stack: flag indicating if projections from serial em stack are used
  • projections: indices of projections that are used, can be
    • None: all projections
    • list: list of projection indices
    • int: randomly pick this number of projections
  • stats: flag indicating if basic image statistics are calculated
  • names_only: flag indicating if instead of projection images the corresponding file names are yielded (useful for testing)
  • translations: flag indication if frame alignment translations are also read

Sets:

  • self.apixel: pixel size in A
  • self.counte: counts per pixel

Yields a dictionary that contains the following (key, value) pairs:

  • 'tilt_angle': tilt angle
  • 'exposure_time': exposure time
  • 'serialem': (pyto.grey.Image) serial em projection image
  • 'original': (pyto.grey.Image) projection frames stack (non-aligned)
  • 'original_flat': (pyto.grey.Image) projection obtained by summing up original (non-aligned) frames without aligning them
  • 'aligned': (pyto.grey.Image) projection obtained from aligned frames
  • 'translations': (ndarray) frame alignment translations

All pyto.grey.Image objects have the following attributes set:

  • mean, std, var, min, max: mean, std, avriance, min, max counts per pixel
  • mean_ea: mean electrons per angstrom

find_counte(self, header)

source code 

Reads number of counts per electron from the specified header.

This header has to be from a frames stack im mrc format generated by DM and SerialEM. Written for Gatan K2 detector.

Argument:

  • header: Frames stack header separated in a list (as attribute pyto.grey.Image.header or pyto.io.ImageIO.header)

Returns number of counts per electron.

read_frame_translations(self, path)

source code 

Reads translations (shifts) between frames of aligned frames image specified by arg path, which was obtained by Dimi, DM or ucsf alignement.

Argument:

  • path: aligned frames image path
  • align: alignment program: 'dm', 'dimi', 'ucsf' or None if shifts should not be read

Returns array of translations (n_frames x 2) or None if the frame alignemend file corresponding to path was not found
